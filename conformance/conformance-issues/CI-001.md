# CI-001: Error Message Format Alignment

**Date:** 2026-07-05
**Status:** RESOLVED
**Test:** TC-009

## Problem

TC-009 (Error Handling) failed in Python and SQLite runtimes due to inconsistent error message format.

- Python: `Resource 'X' not found`
- SQLite: `Resource 'X' not found in database`

## Root Cause

SQLite runtime appended "in database" to the error message, making it inconsistent with Python.

## Resolution

1. Updated SQLite runtime to remove "in database" suffix
2. Both runtimes now produce: `Resource '{id}' not found`
3. Updated TC-009 to reflect the aligned behavior

## Verification

```
Python:  Resource 'non-existent' not found
SQLite:  Resource 'non-existent' not found
```

## Impact

- Python: No change needed
- SQLite: Error message format aligned
- TC-009: Updated expected output
